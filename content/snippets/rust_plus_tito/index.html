Vytvoření RPM balíku přímo z Cargo projektu

November 17, 2016

Co je Cargo

Cargo je správce projektů vytvořených v Rustu. Projekty jsou automaticky přidány do git repozitáře => ideál to zabalit zrovna z toho.

Použité nástroje

Balíky ve Fedoře:

  • tito
  • rpmdevtools
  • mock

Vytvořit nebo vybrat nějaký projekt

např. já jsem forknul Smith: https://github.com/msehnout/Smith

$ git clone <repository>
$ cd <repository>
$ git branch rpm #budu používat pro build v Copr
$ git checkout rpm

Inicializace pro použití s tito

Jednoduše pomocí:

$ tito init

Vytvoření nového spec souboru

$ rpmdev-newspec smith
  1. vyplnit jméno, licenci, závislosti apod.
  2. build bude pomocí cargo => bude potřeba internet na stažení závislostí (většinou z crates.io, případně z gitu). Tady je dobré mít v repozitáři soubor Cargo.lock, který zajistí, že při každém buildu z toho vyleze to samé. Kdyby tam nebyl, tak se můžou změnit verze závislostí: http://doc.crates.io/faq.html#why-do-binaries-have-cargolock-in-version-control-but-not-libraries

PROBLÉM: Verze je tam 2x, jedno ve spec, podruhé v Cargo.toml, co s tím? Nápady:

  • Skript, který bude spouštět tito a přepisovat verzi ve spec file
  • Připsat do tito, aby to umělo samo

Doporučení: Zkontroluj si spec pomocí rpmlint

Lokální testovací build z posledního commitu

Pozor, je to opravdu build z posledního commitu nikoliv pracovního adresáře.

$ tito build --srpm --test`
WARNING: unable to lookup latest package tag, building untagged test project
WARNING: .tito/packages/smith doesn't exist in git, using current directory
Building package [smith-0.3.8-1]
Wrote: /tmp/tito/smith-git-71.b8bcb1a.tar.gz

Build RPM:

Lokální build RPM je možné spustit pomocí:

$ mock -r fedora-24-x86_64 --rebuild /tmp/tito/smith-0.3.8-1.git.71.b8bcb1a.fc24.src.rpm

Nejspíš by to šlo i pomocí tito, ale tam se využívají lokálně instalované balíky, což je problém, protože nepoužívám Rust z repozitářů.

Vytvoření tagu verze

$ tito tag --use-version=$(grep version Cargo.toml | grep -o "[0-9]*\.[0-9]*\.[0-9]*")
$ git push
$ git push --tags

Vytvořit Copr repozitář a balík

Vytvoření repozitáře

  • Potřeba f24 a výš kvůli Rustu
  • přístup k internetu, cargo si bude stahovat všechny závislosti

Balík

  • https odkaz na git
  • větev rpm
  • specifikovat, jestli dělat build z posledního commitu nebo tagu (test)
  • github webhook

Klik na rebuild a už to fičí ☺
